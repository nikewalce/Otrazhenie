# Otrazhenie

Веб-сервис для анализа состава косметики: сканирование штрихкода или ручной ввод состава → разбор по ингредиентам (на основе базы INCI) → оценка безопасности каждого компонента и общий рейтинг продукта.

Статус проекта: **MVP в активной разработке**.

---

## 🔹 1. Что это и зачем

Многие покупатели косметики не понимают состав продукта на этикетке: непонятные названия ингредиентов, неочевидные риски (аллергены, спорные консерванты и т.д.). Otrazhenie решает эту проблему:

- Сканирует продукт по штрихкоду или принимает состав вручную (текстом).
- Разбирает состав на отдельные ингредиенты и для каждого показывает: функцию (консервант, увлажнитель и т.д.) и уровень безопасности.
- Считает общую оценку безопасности продукта.
- Ведёт историю проанализированных продуктов для зарегистрированных пользователей.

## 🔹 3. Функциональность (что уже работает)

### Сканирование и ручной ввод
- Главная страница с формой загрузки изображения штрихкода и формой ручного ввода состава.
- Распознавание штрихкода на изображении через `pyzbar` (`/handle-scan`).
- Ручной анализ состава по введённому тексту (`/manual-analysis`).

### Анализ состава
- Парсинг состава на отдельные ингредиенты с сопоставлением по базе INCI (импортируется из CSV в PostgreSQL).
- Для каждого ингредиента отображаются: название, функция, оценка безопасности, описание.
- Цветовая индикация уровня риска (безопасно / умеренно / риск).
- Автоматическое добавление новой категории ингредиента, если она ещё не известна системе.
- Страница результатов анализа (`/results`).

### Пользователи
- Регистрация и аутентификация (email + пароль), защита форм через CSRF.
- Страница профиля пользователя.

### База данных
- Таблицы: `products`, `product_ingredients`, `ingredient_categories`, модели пользователей.
- Скрипт импорта исходных данных INCI из CSV в PostgreSQL.

## 🔹 6. База данных

**Реализованные таблицы:**

- `users` — пользователи (email, пароль, профиль)
- `products` — продукты, которые были отсканированы/проанализированы
- `product_ingredients` — ингредиенты продукта
- `ingredient_categories` — категории ингредиентов (функция, безопасность)

**Запланированные таблицы:**

- `user_products` — связь пользователь ↔ продукт + дата сканирования (для дневника)
- `product_recommendations` — рекомендации по типу кожи
- `partner_products` — база для подбора безопасных аналогов

## 🔹 7. Roadmap (планы развития)

Краткосрочно (доведение MVP):
- Дневник сканирований и история для пользователя.
- Базовые рекомендации по типу кожи (rule-based, без ML).
- Юнит- и интеграционные тесты на ключевые функции (парсинг, сканирование).
- Очистка данных от языковых префиксов (`en:`, `fr:`) перед показом пользователю.

Среднесрочно:
- Подбор безопасных аналогов продукта по совпадению состава.
- Поиск продукта по названию.
- Админ-панель.
- Локализация интерфейса (RU/EN).

Долгосрочно:
- Мобильное приложение (iOS/Android) на React Native или Flutter, использующее текущий backend как REST API.
- Монетизация по модели freemium (ограничение количества сканирований в бесплатном тарифе, безлимит + расширенные рекомендации в премиум).
- Поддержка GDPR-совместимого хранения данных и удаления аккаунта.

## 🔹 8. Безопасность и ограничение ответственности

- Оценка безопасности ингредиентов носит информационный характер и не является медицинской рекомендацией.
- Хранится только необходимая для работы сервиса информация о пользователе.
- Планируется поддержка удаления аккаунта и явного согласия на сбор данных (GDPR).

## 🔹 9. Установка и запуск (локально)

```bash
# Клонировать репозиторий
git clone https://github.com/nikewalce/Otrazhenie.git
cd Otrazhenie

# Установить зависимости через Poetry
poetry install

# Настроить переменные окружения
cp app/.env.example app/.env  # заполнить значения

# Применить миграции / импортировать данные INCI
poetry run python app/db/csv_to_db.py

# Запустить сервер
poetry run flask run
```

Либо через Docker:

```bash
docker build -t otrazhenie .
docker run -p 5000:5000 otrazhenie
```
